Testing[edit]

Kicksecure ™ needs more people to test new Kicksecure ™ features, bug fixes and so on.

  • Perform numerous activity tests on both Kicksecure ™ and Kicksecure ™, while running a separate, testers-only Kicksecure ™ version based on the stable-proposed-updates APT repository.
  • Test major, new Kicksecure ™ versions prior to stable release - latest testers-wanted posts can be found here: https://forums.kicksecure.com/tags/testers-wanted [archive]
  • Join us in testing new AppArmor profiles for improved security! Visit the Kicksecure ™ AppArmor Forum [archive].

Run the Kicksecure ™ Backup Script[edit]

Create a backup of the Kicksecure ™ Anonymous Operating System website and source code. The more people that retain backups, the less likely Kicksecure ™ can be erased from the history books. This can be done periodically, every now and then or when convenient.

See: How to Backup Kicksecure ™ for further details.

Documentation[edit]

Want to improve Kicksecure ™ wiki, this or any wiki page? Help is welcome and volunteer contributions are happily considered!

Short summary: Anyone can edit the Kicksecure ™ wiki.

Edits are possible with or without account at time of writing.

1. Any huge modification should be discussed in Kicksecure ™ forums beforehand. Smaller changes (such as fixing links, typos, grammars and small enhancements) do not require prior discussion. It is recommended to start with small changes first before taking on major rewrites.

2. There are [[Edit]] buttons next to every wiki chapter.

3. Press the [[Edit]] button.

4. Expand/improve the text or fix links, typos or grammar.

5. Optional: Press the Show preview button.

6. Optional: Press the Show changes button.

7. Further modify the text if required.

8. Save changes.

9. Done. The wiki edit is complete.

10. Wait for moderation to confirm the edit. Should there be any issues, post in Kicksecure ™ forums.

Conditions:

  • Moderation: Edits are held for moderation.
  • Modifications: Please note that all contributions to Kicksecure ™ may be edited, altered, or removed by other contributors. If you do not want your writing to be edited mercilessly and redistributed at will, then do not submit it here.
  • Copyright: You are also promising us that you wrote this yourself. Before copying content here from elsewhere, ask for approval by admins in Kicksecure ™ forums [archive]. Do not submit copyrighted work without permission!
  • Terms: Read, understand and agree to Conditions for Contributions to Kicksecure ™.
  • Policies: Policy of Kicksecure ™ Website and Kicksecure ™ Chat and Policy On Nonfreedom Software applies.

Contributor Responsibilities[edit]

The tasks of a contributor include:

  • Answering questions in the forums.
  • Keeping the wiki up-to-date.
  • Testing of installation, build and upgrade instructions.
  • Following the issue tracker, squashing bugs and perhaps implementing new features relating to the maintained component.
  • Otherwise self-initiated (guess) work and taking care of the component in question.

This is a volunteer position.

Package Contributor[edit]

Kicksecure ™ provides a lot of Debian packages; see Overview [archive] for further information. Due to limited Kicksecure ™ contributors resources, only a few packages are supported on Debian hosts. A lot of these packages are untested and unmaintained in any other Linux distribution.

Would you be interested in maintaining one or more packages for use beyond Kicksecure ™? For example, in Debian, Ubuntu, Fedora or other distributions? Maintenance requirements would involve keeping up with code changes and releases, testing if these packages work outside of Kicksecure ™, eventually reporting bugs or adding new features, and answering user support requests.

If interested, ambitious package contributors could even seek to have one or more Kicksecure ™ packages accepted into a mainstream Linux distribution.

This is a volunteer position.

General Tasks[edit]

People with skills and interest in the following areas would also help:

  • Good knowledge of Linux networking and iptables.
  • AppArmor profile creation and testing of new AppArmor profiles for improved security.
  • #!/bin/bash
  • Programming languages like Python, Ruby, C, and others.
  • Registration at our issue tracker.
  • Participation in the development forum [archive]; consider hitting the "notify" button to subscribe.
  • Discussion participation - login on github and go to https://github.com/Kicksecure/Kicksecure/issues [archive] and press "watch" for notifications about Kicksecure ™ github issues (this is an old issue tracker and less important).

Release Manager[edit]

If you are willing to have your real name connected to the Kicksecure ™ project and Tor, please continue reading.

Depending on your interests, your tasks could include:

  • Building Kicksecure ™ .ova images:
    • Creating stable releases.
    • Creating stable hotfix releases.
    • Creating assorted other builds. [5]
  • Building Kicksecure ™ .libvirt images.
  • Reporting bugs when builds fail.
  • Signing and uploading releases.
  • Updating the download page.
  • Posting to the blog or mailing lists.
  • Fixing simple build documentation.
  • Dev/Redistribution
  • Testing. [6]

This is a volunteer position.
