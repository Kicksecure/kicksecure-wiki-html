Special:Preferences - mediawiki developer documentation[edit]

Under Special:Preferences#mw-prefsection-rendering a few months time ago, dev requested the following settings to be disabled. 

Skin
    ...
	Vector legacy (2010) (Preview)

Skin preferences
	Enable responsive mode
    Adapt layout to screen size on mobile.

This was a long standing todo item for admin. Wasn't easy to resolve.

This is because these other skin and the responsive mode would be hard to make compatible with our custom skin.

Admin now figured out how to do that. Here are the MediaWiki settings to do that. 

## Under ../Special:Preferences#mw-prefsection-rendering remove:
##    Enable responsive mode
##    Adapt layout to screen size on mobile.
$wgHiddenPrefs = array( "skin-responsive", );

## Under ../Special:Preferences#mw-prefsection-rendering remove:
##    Vector legacy (2010) (Preview)
##
## hide 'Vector legacy (2010)'
## keep 'Vector 2022'
$wgSkipSkins = [
    'vector',
];

Please document this on Dev/mediawiki. (Because maybe one day in the future we'd wonder why we don't have these settings available.)

Install VirtualBox locally[edit]

Using Ubuntu 22.10 (kinetic)? Try installation of the VirtualBox host software locally. Installation should now be simple. Should even be compatible with SecureBoot. Was tested by a contributor (though the testing was in a VM, therefore a bit limited). Please try to install. Instructions: Template:VirtualBox_Host_Software_Installation

DEV

  • Virtualbox was installed successfully on a Xubuntu machine with Secure Boot
  • Whonix-Gateway and Whonix-Workstation were launched successfully
  • However the installation was not possible with the Software center
  • The installation worked with sudo apt install virtualbox-dkms. This is due to an installation step where you are asked to enter a boot password. And after rebooting with this boot password you "enroll keys" in the boot process. Then virtualbox works fine. If you just install the package virtualbox this step is skipped and virtualbox will start but will not run any virtual machines due to some kernel drivers missing
  • Also I had to activate VT-x in the Boot menu to enable virtualbox to function properly ../VirtualBox/Troubleshooting#VT-x_is_disabled_in_BIOS
  • After that Whonix-Gateway and Whonix-Workstation both ran correctly
